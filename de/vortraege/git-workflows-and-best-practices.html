Git

Workflows and best practices

Kurzbeschreibung

Bereits seit 2010 nutze ich git bei der täglichen Arbeit. In diesem Vortrag möchte ich die Best Practices von git zeigen. Es wird um die "lessons learned" gehen, genauso wie um das Thema destruktive Kommandos und warum man sie nicht nutzen sollte. Des Weiteren werde ich auf branching Modelle eingehen und warum eine Abwandlung von "git flow" für mich bis heute am besten funktioniert. Dabei werde ich auch auf die Integration von git in Ticket Systeme eingehen.
