Git

Workflows and best practices

Talk Abstract

Since 2010 I’m using git on a daily base. In this talk I want to show my best practices in using git commands. I’ll talk about lessons I learned and how my view is on most discussed topic whether and when to use history changing commands. On top I’d like to give an introduction of a branching model close to git flow and how it can help to get your development life-cycle in a better shape including integrating git into ticket systems.

Talk Details

  • Length: up to 120 minutes
  • First held: March 2017
  • Level: Intermediate
