99%

is not enough

Kurzbeschreibung

Jeder stimmt zu, dass Unit Tests essentiell wichtig sind um, robuste und wartbare Software zu erstellen. Aber wie viel Code Coverage sollte ein Projekt haben? Wir behaupten, dass selbst 99% Abdeckung nicht genug sind.

Wir möchten euch zeigen, wie wir Tests erstellen, was wir tun müssen um, unseren Sourcecode testbar zu machen und warum das erreichen der 100 prozentigen Abdeckung unseres Codes so wichtig für uns ist.

Zielgruppe

Dieser Vortrag richtet sich an PHP Entwickler und Software Architekten denen es wichtig ist, wartbaren und refakturierbaren Programmcode zu erstellen. Grundkenntnisse über die Erstellung von Tests und was Code Coverage bedeutet, sind von Vorteil.
